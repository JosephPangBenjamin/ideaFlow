# Product Brief: IdeaFlow

## Executive Summary

**IdeaFlow** 是一款将**想法捕捉**与**任务执行**无缝融合的效率工具。它解决了一个普遍但被忽视的问题：人们的灵感和想法散落在微信、截图、笔记等各种地方，随着时间推移逐渐被遗忘，即使偶然翻到也无法回忆起当时的场景和初衷。

IdeaFlow 通过**可视化自由画布**、**上下文保留**和**AI智能分析**，让每一个想法都能被追踪、演进和执行，帮助用户从"碎片化记录者"转变为"有条不紊的执行者"。

---

## Core Vision

### Problem Statement

现代知识工作者面临严重的**想法碎片化**问题：
- 灵感随机出现，被匆忙记录在各种工具中（微信、截图、笔记App）
- 记录越多，信息越沉底，逐渐被遗忘
- 即使重新发现，也无法回忆当时的想法、场景和记录原因
- 想法捕捉和任务执行使用不同工具，缺乏连贯性

### Problem Impact

- 有价值的想法被埋没，错失执行时机
- 重复思考相同问题，浪费认知资源
- 无法追踪想法的演进历程，失去迭代优化的机会
- 个人效率受限，创意产出降低

### Why Existing Solutions Fall Short

| 工具类型 | 局限性 |
|----------|--------|
| 微信文件传输助手 | 仅存储，无法组织、追踪、提醒 |
| 传统 Todo 工具 | 只管任务执行，不保留想法上下文 |
| 笔记工具（Notion等） | 结构化要求高，不适合快速碎片捕捉 |
| 思维导图工具 | 擅长规划，弱于执行追踪 |
| 截图/收藏 | 完全碎片化，无法激活和转化 |

**核心差距**：没有工具能同时做到 **快速捕捉 + 上下文保留 + 可视化规划 + 进度追踪 + AI辅助回忆**。

### Proposed Solution

IdeaFlow 提供一体化的想法生命周期管理：

1. **闪念捕捉**：随时随地快速记录想法，自动附加时间戳和来源上下文
2. **可视化画布**：自由拖拽、连线标注，像思维导图一样组织想法关系
3. **任务转化**：一键将想法转为可执行任务，设置时间、优先级、分类
4. **进度追踪**：时间线视图 + 状态管理，清晰掌握每个想法的执行进度
5. **AI 智能助手**：
   - 定期智能汇总
   - 触发条件提醒
   - 主动推荐回顾
   - 分析任务关联
6. **轻量协作**：偶尔分享给团队、@成员分配任务

### Key Differentiators

| 差异化 | 描述 |
|--------|------|
| 🧠 **上下文保留** | 自动记录"为什么记"，帮助未来的自己恢复记忆 |
| 🎨 **自由画布** | 像蓝湖一样缩放平移，像Word一样拉线标注 |
| 🔄 **一体化** | 从想法到任务无缝衔接，不再切换工具 |
| 🤖 **AI记忆辅助** | 智能浮现沉底想法，预测任务关联 |
| 📊 **双重进度追踪** | 时间线（什么时候做什么）+ 状态（做到哪了） |

## Target Users

### Primary Users

#### 🎓 小明 - 大学生创业者

**背景**：大三学生，商科专业，脑子里总有各种创业点子

**核心特征**：**点子多，但执行少**

**典型一天**：
- 上课时突然想到某个App点子，赶紧记在手机备忘录
- 刷抖音看到有趣的商业模式，截图保存到相册
- 躺床上又想到怎么改进，但懒得起来记录
- 周末想开始做某个点子，却发现想法碎片散落各处，不知从何下手

**痛点**：想法太多无法聚焦，缺乏执行的推动力和追踪机制

**期望**：一个工具帮我**从"想"到"做"**，而不只是更好地记录

---

#### 💼 小红 - 职场产品经理

**背景**：工作3年的产品经理，负责多个项目并行推进

**核心特征**：**想法碎片化，执行需协作**

**典型一天**：
- 开会时领导提到某个需求方向，赶紧记在笔记本上
- 通勤时刷到竞品新功能，微信发给自己"待研究"
- 需要分配任务给设计师和开发时，发现之前的想法找不到了

**痛点**：想法散乱无法形成体系，协作时无法让他人快速理解上下文

**期望**：**整理好的想法可以直接变成可分配的任务**，@同事后对方秒懂

---

### Secondary Users

#### 👥 项目协作者

**身份**：被@的团队成员（必须注册账号）

**场景**：收到点子分享后觉得不错，加入协作

**需求**：
- 能看到完整的想法上下文（清晰的公开视图）
- 能添加自己的补充和标注
- 能追踪整个项目的进度

**价值**：从被动接收任务变为主动参与共创

## Success Metrics

### 🌟 North Star Metric（北极星指标）

**周活跃执行用户数（Weekly Active Executors, WAE）**

> 定义：每周至少**完成1个任务**的用户数

**选择理由**：
- ✅ 直接衡量核心价值"推动点子落地"
- ✅ 不易被刷数据
- ✅ 连接用户价值→商业价值

---

### 📊 Diagnostic Metrics（诊断指标）

| 指标 | 定义 | 作用 |
|------|------|------|
| **点子→任务转化率** | 想法变成任务的比例 | 诊断"规划"环节 |
| **任务完成率** | 任务从创建到完成的比例 | 诊断"执行"环节 |
| **沉底点子激活数** | 被重新激活的7天+旧点子数 | 诊断"AI提醒"价值 |

---

### ⏰ Key Definitions（关键定义）

| 术语 | 精确定义 |
|------|----------|
| **沉底点子** | 创建后 ≥7 天未被编辑或转为任务的点子 |
| **激活** | 对沉底点子执行"编辑"或"转为任务"操作 |
| **活跃执行用户** | 本周内完成 ≥1 个任务的用户 |

---

### 📈 MVP 埋点方案

| 事件 | 触发时机 | 记录数据 |
|------|----------|----------|
| `idea_created` | 创建点子时 | idea_id, user_id, timestamp |
| `idea_viewed` | 打开点子详情时 | idea_id, user_id, timestamp |
| `idea_edited` | 编辑点子内容时 | idea_id, user_id, timestamp |
| `idea_to_task` | 点子转为任务时 | idea_id, task_id, user_id, timestamp |
| `task_completed` | 任务标记完成时 | task_id, user_id, timestamp |

## MVP Scope

### 📋 Release Strategy（发布策略）

采用**三阶段发布**策略，快速验证核心价值：

---

### 🚀 Phase 1: Core Value（核心价值验证）- 2-4周

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 手机号注册/登录 | 用户身份基础 | P0 |
| 快速创建想法 | 闪念捕捉 | P0 |
| 想法→任务转化 | 推动执行的核心 | P0 |
| 任务时间设置 | 创建/修改/完成时间追踪 | P0 |
| 进度追踪（状态管理） | 看到执行结果 | P0 |
| 任务分类/模块 | 基础组织能力 | P0 |

**Phase 1 目标**：验证"点子→任务→完成"的核心价值链

---

### 🎨 Phase 2: Differentiation（差异化体验）- +2-3周

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 自由画布 | 拖拽/缩放/平移，类蓝湖体验 | P1 |
| 拉线标注 | 可视化关联，类Word批注 | P1 |
| 公开/私密视图切换 | "私密时自由，公开时体面" | P1 |

**Phase 2 目标**：建立差异化竞争优势

---

### 👥 Phase 3: Collaboration（协作能力）- +2-3周

| 功能 | 描述 | 优先级 |
|------|------|--------|
| @成员分配任务 | 协作核心功能 | P2 |
| 团队协作/共享 | 多人共同编辑 | P2 |

**Phase 3 目标**：扩展到团队使用场景

---

### 🔜 Future: AI Enhancement（AI增强）- 后续版本

| 功能 | 描述 | 替代方案 |
|------|------|----------|
| AI 智能总结 | 自动总结任务 | MVP暂不实现 |
| AI 沉底提醒 | 7天未操作提醒 | 定时任务+邮件通知 |

**决策**：AI功能简化/推迟到后续版本，先验证核心价值

---

### 🛠️ Development Standards（开发规范）

| 规范 | 要求 |
|------|------|
| **开发方法** | TDD（测试驱动开发） |
| **测试覆盖** | 核心功能100%覆盖 |
| **埋点设计** | MVP即包含数据追踪能力 |

---

### ✅ MVP Success Criteria（成功标准）

**Phase 1 发布后验证**：
- 用户能够完成"创建想法→转为任务→标记完成"的完整流程
- 周活跃执行用户数（WAE）达到预期
- 用户留存率（7日）≥ 40%

**扩展决策点**：
- Phase 1 验证成功 → 继续 Phase 2
- 核心流程有问题 → 迭代优化 Phase 1

---

### 🚫 Out of Scope for MVP（MVP不包含）

| 功能 | 理由 | 计划时间 |
|------|------|----------|
| 移动端App | 先验证Web端价值 | V2.0 |
| 第三方登录 | 手机号已满足需求 | V1.5 |
| 高级AI功能 | 需要更多数据积累 | V2.0 |
| 导入/导出功能 | 非MVP核心 | V1.5 |
| 多语言支持 | 先专注中文市场 | V2.0 |
